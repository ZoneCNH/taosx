# taosx 身份

## 我是谁

`taosx` 是 FoundationX 的 **TDengine 存储扩展模块**，提供 TDengine 时序数据库客户端封装。

## 我做什么

- TDengine 连接管理和健康检查
- 时序数据写入/查询接口封装
- 超级表/子表管理

## 我不做什么

- 不是时序业务逻辑 — 数据语义由调用方定义
- 不是模板源 — 模板生成属于 xlib-standard
- 不依赖其他存储模块

## 与 clickhousex 的时序存储边界

`taosx` 与 `clickhousex` 都处理时序数据，但面向不同场景，互补不重叠：

| 维度 | taosx | clickhousex |
|------|-------|-------------|
| 存储引擎 | TDengine | ClickHouse |
| 面向场景 | **IoT 时序存储** | **OLAP 分析查询** |
| 典型负载 | 高频写入、时间窗口查询、设备/行情数据 | 聚合查询、多维分析、即席查询 |
| 数据特点 | 流式写入、超级表、按时间降采样 | 宽表、星型模型、历史归档 |
| 选型依据 | 需要高吞吐写入和简单窗口查询的场景 | 需要灵活聚合和快速即席分析的场景 |

**选型指南**：
- 高频写入 + 简单时间窗口查询 → `taosx`（TDengine 原生时序优势）
- 复杂聚合 + 多维即席分析 → `clickhousex`（ClickHouse OLAP 优势）
- 需要历史数据归档和星型模型分析 → `clickhousex`
- 需要毫秒级时序流式写入和降采样 → `taosx`

两者同属基座存储扩展层，功能互补，不在同一维度竞争。
